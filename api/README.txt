NP.Maps — серверная заявка (api/lead.php)

Зачем это нужно
- Это запасной путь получения заявок (помимо Telegram/копирования/звонка).
- Самый надежный вариант на большинстве хостингов — Telegram Bot.

Быстрый старт (рекомендуется: Telegram)
1) Создайте Telegram-бота через @BotFather и получите токен.
2) Узнайте chat_id:
   - Напишите боту
   - Или добавьте бота в чат/канал и используйте @userinfobot / getUpdates (зависит от схемы)
3) Скопируйте api/config.sample.php → api/config.php
4) Заполните:
   - 'tg_bot_token'
   - 'tg_chat_id'
5) Загрузите сайт на хостинг с PHP.

Email (fallback)
- Заполните 'to_email' в api/config.php
- Учтите: на части хостингов функция mail() отключена. Тогда используйте Telegram-вариант или SMTP-решение.

Безопасность
- lead.php принимает только POST + JSON.
- Все заявки можно логировать в api/leads.jsonl (по умолчанию включено).

Проверка
- Если не настроено ни одного канала доставки, API вернет:
  {"ok":false,"error":"not_configured"} (HTTP 503)


Быстрый вариант без поиска числового chat_id (рекомендуется)
- Создайте приватный канал, задайте ему @username (например @np_maps_leads)
- Добавьте бота в канал и назначьте админом (разрешите постить)
- В api/config.php укажите:
  'tg_chat_id' => '@np_maps_leads'

Так вы можете отправлять заявки, не вытаскивая числовой id.
